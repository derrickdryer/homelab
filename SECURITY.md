# Security Policy

Security is taken seriously in this project. If you discover a potential security vulnerability or have concerns related to the security of this repository, please report it responsibly. Thoughtful and coordinated disclosure is appreciated, and every reasonable effort will be made to review and address valid reports.

---

## Reporting a Vulnerability

To report a security issue, please use GitHub’s built-in security reporting feature:

1. Navigate to the **Security** tab of this repository.
2. Select **Report a vulnerability**.
3. Provide a clear and detailed description of the issue and its potential impact.
4. Include steps to reproduce the issue or a Proof of Concept (PoC), if applicable.
5. Submit the report.

Once a private report is received, the issue will be reviewed and assessed as promptly as possible.

Please **do not publicly disclose** security vulnerabilities through issues, pull requests, forums, or social media until the matter has been reviewed and addressed.

---

## Scope

This security policy applies to all code, configurations, and documentation contained within this repository that are actively maintained by the author.

If a security issue is identified in a third-party dependency or external project not maintained here, please follow that project’s responsible disclosure process and notify the appropriate maintainers.

---

## Expectations & Limitations

This repository is maintained as part of a personal homelab and learning environment. While reasonable care is taken to follow security best practices, response times and mitigation options may be limited compared to large organizations or commercial projects.

Thank you for helping improve the security and quality of this project.
